NARF tester
===========

The tester is a small interactive command-line program for exercising NARF disk
images.

Building
--------

From the `src` directory:

```
make
```

This builds `tester` or `tester.exe`, depending on the host environment and
Makefile target naming.

Invoking
--------

```
./tester <image>
./tester =<size>,<image>
```

Examples:

```
./tester test.img
./tester =16M,test.img
./tester =512K,tiny.img
```

If the argument starts with `=`, the tester's mmap I/O layer creates the image
when it does not already exist.  Size suffixes `K`, `M`, and `G` are accepted by
the example I/O layer.

After startup, the prompt is:

```
#>
```

Use `help` to print a compact command summary: the first line of each command's
help text, several per line, separated by `|`.  Use `help <command>` to print
that command's detailed help string.  Use `quit` or `exit` to leave.
An empty line prints `huh? Empty command.` and the same compact help summary.

Common setup
------------

For an MBR-partitioned test image:

```
mbr
partition 1
format 1
mount 1
```

For a whole-image filesystem:

```
mkfs
init
```


Command parser notes
--------------------

Tester commands are registered in a table containing the command name, an
`argc`/`argv` style handler function, and the help text used by `help` and
`help <command>`.  Command names must match exactly; accidental prefix matches
such as `debugger` no longer dispatch to `debug`.

Arguments are split roughly like a tiny shell: whitespace separates arguments,
double quotes keep spaces inside one argument, and `\n`, `\r`, `\t`, `\\`, and
`\"` are decoded.  Commands such as `create`, `append`, `tag`, and `mbr` join
remaining arguments back together with spaces, so both of these work:

```
tag foo "v1 uid=1000 gid=1000 mode=100644 mtime=1778706214 bs=4K"
tag foo v1 uid=1000 gid=1000 mode=100644 mtime=1778706214 bs=4K
```

Commands
--------


### `help [command]`

With no argument, print a compact command summary generated from the tester's
command table.  It uses the first line of each command's help text, wrapped
across several `|`-separated entries per line.  With a command name, print that
command's detailed help string.

Examples:

```
help
help tag
help gremlins
```

Entering an empty line at the tester prompt is treated as an empty command:
it prints `huh? Empty command.` followed by the compact help summary, then
returns to the prompt.

### `mbr [message]`

Write a classic MBR to sector 0.  With no message, the default boot message is
used.  With a message, the provided text is embedded in the boot-code area.

### `partition <number>`

Create a NARF partition entry.  Valid partition numbers are 1 through 4.

### `findpart`

Search the MBR for a NARF partition and print the partition number found.

### `format <number>`

Format the selected NARF partition.

### `mount <number>`

Mount the selected NARF partition.

### `mkfs`

Format the whole image as a NARF filesystem starting at sector 0.

### `init`

Mount/initialize a whole-image filesystem starting at sector 0.

### `alloc <key> <bytes>`

Create a new key with the requested byte size.  The initial payload is
zero-filled.

Example:

```
alloc empty.bin 1024
```

### `create <key> <string>`

Create a new key and initialize it with string data.  Quoted strings are
accepted.  Inside quotes, the tester understands `\n`, `\r`, `\t`, `\\`, and
`\"`.

Examples:

```
create hello.txt "hello from NARF"
create lines.txt "one\ntwo\n"
```

### `append <key> <string>`

Append string data to an existing key.  Quoted strings and the same simple
escapes used by `create` are accepted.

Example:

```
append hello.txt " plus more"
```

### `realloc <key> <bytes>`

Resize an existing key.  Growing a file exposes zero-filled bytes.  Shrinking
reduces the byte size and may release storage according to the current core
implementation.

### `rename <old-key> <new-key>`

Rename a key.

### `free <key>`

Delete a key and return its storage to the filesystem.

### `ls <dirname>`

List keys that appear directly under `dirname` using `/` as the separator.
NARF has no real directories; this is prefix/separator traversal.
Keys normally do not start with `/`; root can be listed as `/` as a convenience.

Examples:

```
ls /
ls /dir/
```

### `cat <key>`

Print a hex/ASCII dump of a key's payload.

### `tag <key> <text>`

Store a metadata string in the key's metadata area.  Quote the text when it
contains spaces.

Example:

```
tag foo "v1 uid=1000 gid=1000 mode=100644 mtime=1778706214 bs=4K"
```

### `scan <key>`

Read and print the key's metadata area as a string.

### `pack <host-directory>`

Recursively copy files from a host directory into NARF.  Keys are generated from
relative paths under the host directory.  Directory keys end in `/`.

Example:

```
pack /tmp/narfpack/
```

### `slurp <host-file>`

Read line-oriented keys from a host text file and allocate each key with 1024
bytes.  This is mostly a stress-test helper.

### `defrag`

Call `narf_defrag()`.  When `NARF_USE_DEFRAG` is enabled, this compacts payload
extents by moving data into lower free holes using small copy-on-write catalog-record
commits.  It also merges adjacent free extents, lowers the data frontier when
the highest payload extent becomes free, and reclaims zero-length record-free
nodes from the top of the record area.

### `debug`

Print internal root/data-tree/free-tree/index-tree information.  This requires `NARF_DEBUG` to
be enabled in the build.

### `gremlins <seed> <count>`

Run randomized tester operations.  The seed makes a run reproducible.  The count
controls how many random operations are executed.

Example:

```
gremlins 1234 100
```

Example session
---------------

```
mbr
partition 1
format 1
mount 1
create hello.txt "hello from NARF"
append hello.txt "\nmore text"
cat hello.txt
ls /
gremlins 1234 50
debug
quit
```
